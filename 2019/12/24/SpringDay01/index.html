    Spring框架

    1. 由一系列组件组成的框架,是软件的半成品,包含了软件常用的组件,可以作为软件的基石快速搭建开发软件
    2. 其核心功能: IOC(控制反转) AOP(面向切面儿编程)
    3. 使用Spring必须遵守Spring约定的规则

    IOC控制反转

    1. 控制反转: 相对主动控制对象而言,将对象的控制交给容器管理的现象叫控制反转

    2. Spring核心功能IOC: 可以帮助我们管理对象,使用者只需要从Spring得到对象,然后使用对象.

    3. 测试Spring IOC功能:

 3. 编写配置文件: 将类名告诉SPring
 4. 启动Spring,Spring会自动创建管理对象
 5. 通过Spring获得对象,测试对象

    Java Bean

    • Spring建议被管理的对象时JavaBean对象,Spring可宽泛的支持任何Java对象

    • Java Bean: 符合一定规范的Java对象

        1. 有Package
  2. 有无参数构造器
  3. 实现序列化接口
  4. 可以使用gatXxx/setXxx定义"Bean"属性

      > 可以利用开发工具自动生成

    • Bean属性: 是指getXxx和setXxx, bean属性名xxx

    • 对象属性: 是指对象的实例变量

      例子:

      class Person{
  String name; //实例变量,对象属性
  String getName(){ // Bean属性,属性名 name
      return name;
  }
}

    Spring IOC容器提供了丰富的Bean管理功能

    • Spring为了支持任何软件,提供丰富的Bean对象管理功能

    • 单例

    • 单例: 软件运行期间只有唯一不变的一个实例
    • Spring默认情况下按照单例规则管理对象

      例子:

      public class TestCase{
  ClassPathXmlApplicationContext ctx;
  @Before //在...之前,在测试案例之前
  public void init(){
      System.out.println("init");
      ctx = new ClassPathXmlApplication("applicationContext.xml");
  }
  @Test
  public void testDemoBean(){
      DemoBean bean = (DemoBean)ctx.getBean("demo");
      System.out.println(bean);
  }
  @Test
  public void testSingleton(){
      DemoBean bean1 = (DemoBean)ctx.getBean("demo");
      DemoBean bean2 = (DemoBean)ctx.getBean("demo");
      System.out.println(bean1==bean2);
  }
  @After //在...之后,在测试案例之后
  public void destroy(){
      System.out.println("destroy");
      ctx.close();
  }
}
