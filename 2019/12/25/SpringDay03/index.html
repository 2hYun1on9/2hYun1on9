    关于Spring的概念

    对象容器: 集合
    Web容器,Servlet容器:Java Web服务器

    注解

    Spring支持注解方式声明Bean组件,这种方式的好处是可以利用编译期对注解进行检查,减少错误的繁盛

    利用注解创建Bean组件

    1. 创建项目,导入Spring
      <dependencies>
 <dependency>
     <groupId>junit</groupId>
     <artifactId>junit</artifactId>
     <version>4.12</version>
   </dependency>
   <dependency>
       <groupId>org.springframework</groupId>
       <artifactId>spring-context</artifactId>
       <version>4.3.25.RELEASE</version>
   </dependency>
       <dependency>
     <groupId>mysql</groupId>
     <artifactId>mysql-connector-java</artifactId>
     <version>5.1.6</version>
 </dependency>
 <dependency>
     <groupId>commons-dbcp</groupId>
     <artifactId>commons-dbcp</artifactId>
     <version>1.4</version>
 </dependency>
</dependencies>
    2. 在配置文件中启动组件扫描
      <!-- 开启组件扫描功能:Spring会递归扫描cn.tedu.beans包中的全部类,查找相关注解,如果找到就将注解标注的类创建为Java Bean对象 -->
<context:component-scan base-package="cn.tedu.beans" />
    3. 在被扫描到的包中声明类!
      @Component
public class DemoBean{
 public String toString(){
     return "Hello World";
 }
}
    4. 测试
      public class TestCase{
 ClassPathXmlApplicationContext ctx;
 @Before
 public void init(){
     ctx = new ClassPathXmlApplicationContext("applicationContext.xml");
 }
 @Test
 public void testDemoBean(){
     // 测试利用注解声明的Bean组件
     DemoBean bean = ctx.getBean("demoBean",DemoBean.class);
     System.out.println(bean);
 }
 @After
 public void destroy(){
     ctx.close();
 }
}

    组件

    • Spring注解,用于标注在类上,被Spring扫描到以后Spring会将类实例化为一个Bean对象,并自动标注BeanID为”demoBean”

    常用组件

    @Component 组件,用于标注通用组件
    @Repository 仓储,用于标注存储组件
    @Service 服务,用于标注业务功能组件
    @Controller 控制器,用于标注控制器功能组件
    @Lazy 懒惰加载
    @Component(“”) 自定义ID(创建BeanID别名)
    @Scope(“prototype”) 多例模式,如果不写就是单例模式
    @Autowired 按照类型自动注入注解
    功能相同,但是”语义”不同,建议按照语义使用,目的是写出”人能看懂的代码”!

    懒惰加载

    默认时候组件采用立即初始化方式,也就是Spring容器启动时候立即创建Bean对象,标注@Lazy注解

    案例:

    1. 声明类
      @Component
@Lazy
public class DemoBean{
 public DemoBean(){
     System.out.println("init DemoBean()");
 }
 public String toString(){
     return "Hello World";
 }
}
    2. 测试
      @Test
public void testLazy(){
 // 懒惰方式测试
 System.out.println("OK");
 DemoBean bean = ctx.getBean("demoBean",DemoBean.class);
}

    多个实例

    Spring默认情况下组件时单例的,利用注解@Scope("prototype")时则创建多个实例

    案例:

    1. 声明类:
      @Component
@Scope("prototype");
public class ExapleBean{
 public String toString(){
     return "ExapleBean";
 }
}
    2. 测试:
      @Test
public class testExampleBean{
 // @Scope("prototype")
 ExampleBean example = ctx.getBean("example",ExampleBean.class);
}

    自定义组件ID

    案例:

    1. 声明类
      @Component("myBean")// 自定义组件ID
public class TestBean{
 public String toString(){
     return "TestBean";
 }
}
    2. 测试
      @Test
public void testMyBean(){
 // Component("myBean"); 自定义组件ID
 TestBean bean = ctx.getBean("bean",TestBean.class);
 System.out.println(bean);
}

    DI

    Spring 提供了自动注入注解 @Autowired
    首先按照类型自动注入,如果有多个类型相同,再按照ID和名称进行匹配
    原理:

    案例:

    1. 创建Bean组件:
      public interface Tool{

    }

    @Component
    public class Axe implements Tool,Serializable{
    private String name=”斧子”;
    public String toString(){
    return name;
    }
    }

    @Component(“tool”)
    public class Saw implements Tool,Serializable{
    private String name=”电锯”;
    public String toString(){
    return name;
    }
    }

    @Component
    public class Worker implements Serializable{
    private String name=”光头强”;
    @Autowired
    private Tool tool;
    public void work(){
    System.out.println(name+”使用”+tool+”砍树”);
    }
    }

    2. 测试:

    @Test
    public void testWorker(){
    // 测试自动注入功能
    Worker worker = ctx.getBean(“worker”,Worker.class);
    worker.work();
    }
